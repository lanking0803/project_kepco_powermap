# KEPCO Web Map — 초기 설계 계획서

> 작성일: 2026-04-08 | 상태: Phase 1 구현 완료
> 현재 DB 스키마는 마이그레이션 파일(`db/migrations/`)이 정확한 소스.

---

## 1. 목표

- **Supabase 단일 진실 소스(SoT)** 기반 웹 지도 서비스
- 관리자가 KEPCO 엑셀 업로드 → DB에 누적 → 모든 사용자가 지도 조회
- 지오코딩 결과 영구 캐시 (재호출 방지)
- 인증 + 권한 분리 (admin / viewer)

---

## 2. 핵심 데이터 흐름

### 업로드 (관리자만)
> **2026-04-22 폐기**: 엑셀 업로드 기능 자체 비활성화. 아래는 초기 설계 기록.
```
엑셀 선택 → 양식 검증 → 파싱 + 행 검증 → 파일 내 중복 제거
→ 지오코딩 → DB UPSERT → MV REFRESH
```

### 조회 (모든 인증 사용자)
```
페이지 진입 → 인증 체크
→ /api/map-summary (Light, 리 단위 집계)
→ 지도 + 마커 + 클러스터
→ 마커 클릭 → /api/location (Heavy, 해당 리 raw 데이터)
```

---

## 3. 인증 / 권한

- **Supabase Auth** (이메일+비번), 회원가입 UI 없음 (관리자가 직접 발급)
- **admin**: 업로드, 데이터 삭제, 계정 관리, 크롤링 제어
- **viewer**: 조회만 (지도/필터/검색/거리재기 등)

---

## 4. 양식 검증 정책

### KEPCO 엑셀 양식
- Row 0~1: 타이틀 + 그룹 헤더 (병합셀)
- **Row 2**: 컬럼 헤더 (21개)
- Row 3~: 데이터 행

### 검증 단계
1. **파일 검증**: .xlsx/.xls, 첫 시트만
2. **헤더 자동 탐지**: 1~10행 중 "시/도" 포함 행. 필수 21개 + 선택 STEP 6개.
3. **행 검증**: 빈 행 스킵, 주소/시설 누락 스킵+카운트, 비숫자 0 처리
4. **파일 내 중복**: 9개 키 조합, 마지막 행 우선

## 5. 합의된 결정 사항

| 항목 | 결정 |
|---|---|
| 데이터 저장소 | Supabase (단일 진실 소스) |
| 인증 | Supabase Auth, 회원가입 X |
| 권한 | admin / viewer 2단계 |
| 지오코딩 | ~~카카오 메인 + VWorld fallback~~ → **VWorld 단독 + PNU 동시 획득** (2026-04-22 변경) |
| 캐시 | ~~geocode_cache~~ → `kepco_addr` 단일 저장 + 지번은 Vercel KV (2026-04-22 변경) |
| 데이터 정책 | upsert (마지막 우선) |
| Upsert 키 | row_hash (9개 조합의 MD5) |
| 2단계 로딩 | Light(map_summary) + Heavy(location) |
| "-기타지역" | 렌더 시 필터 |
