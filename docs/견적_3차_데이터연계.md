## 🔌 데이터 소스별 실제 연계 스펙

### 1. 전국태양광발전소전기사업허가정보 (요청 1 핵심)

- **Endpoint**: `https://api.data.go.kr/openapi/tn_pubr_public_solar_gen_flct_api`
- **공공데이터포털**: data.go.kr/data/15107742
- **승인**: 자동승인, 운영계정 전환 완료(2026-04-25), 만료 2028-04-25
- **응답 필드 (실측 17개, camelCase)** — 명세 PDF 의 대문자 표기와 다름:
  ```
  solarGenFcltNm (시설명), lctnLotnoAddr (지번주소), lctnRoadNmAddr (도로명),
  latitude / longitude, instlDtlPstnSeNm (설치위치 — 지상/옥상/기타),
  capa (kW), oprtngSttsSeNm (가동상태), splyVolt, freq,
  instlYr (설치년도), instlArea (설치면적 ㎡), detlsUsg (상세용도),
  prmsnYmd (허가일), prmsnInst (허가기관), crtrYmd (기준일),
  insttCode / insttNm (제공기관)
  ```
- **매칭 방식**: 같은 지번 OR 좌표 50m 반경 → "☀️ 설치완료" 표시
  - 의뢰자 의도 (2026-04-26): "이미 설치된 것도 그대로 두고 그 주변 물건으로 영업"
  - 즉 회색/숨김 X. 표시만 추가, 영업 여부는 본인 판단
- **⚠️ 중대한 API 한계 (2026-04-26 직접 호출 검증)**:
  - **검색 필터 전부 미지원** — LCTN_LOTNO_ADDR / LCTN_ROAD_NM_ADDR / SOLAR_GEN_FCLT_NM 모두 NODATA. 응답에 있는 값을 정확히 입력해도 작동 안 함
  - LATITUDE / LONGITUDE → byte 단위 정확 일치만 (실용성 0)
  - **유일하게 작동하는 입력 = pageNo + numOfRows + type** (페이지네이션만)
  - 따라서 사용자 클릭 시점 외부 API 호출 = 불가
  - **실제 구현 = 12만건 전수 batch 다운로드 → Supabase `solar_permits` 테이블 적재 → PostGIS `ST_DWithin(geom, point, 50m)` 로 우리 DB 검색**
  - 본 견적 (① 150만 / 3~4주) 에 위 batch 수집기 + DB migration + by-pnu/near-point endpoint 모두 포함
  - 상세: [.claude/memory/reference_solar_permit_api.md](../.claude/memory/reference_solar_permit_api.md)
- **데이터 한계** (이건 그대로):
  - 3kW 이하 자가발전 누락 (허가 대상 아님) — 축사/온실은 대부분 30kW↑라 영향 적음
  - 허가만 받고 미시공 케이스 → `oprtngSttsSeNm` 필드로 필터링 가능

### 2. 국토부 도로(현황) — 이격거리 도로 판정용

- **공공데이터포털**: data.go.kr/data/15125057
- **형태**: SHP 파일 월간 다운로드 (전국 12,300건)
- **무료, 자동승인 여부 불명** (추후 확인)
- **차로수 필드** 존재 여부 미검증 → **작업 전 SHP 다운로드해서 속성 확인 필수**
- **대안**: 도로중심선 (data.go.kr/data/15146878) 에 차선수 포함
- **필드 없으면**: 도로등급(국도/지방도/시도) 로 근사 (정확도 80%)
- **DB 저장**: Supabase PostGIS 로 import → 공간 쿼리

### 3. 법제처 자치법규 조회 API — 조례 수집

- **Endpoint**: `http://open.law.go.kr/LSO/openApi/`
- **공공데이터포털**: data.go.kr/data/15058294
- **포맷**: XML, 조문별 전문 제공
- **자동승인, 무료, 실시간 업데이트**
- **받아올 수 있는 것**:
  - 조례 제목, 조문 전문, 시행일, 개정이력
  - 예: "여수시 도시계획 조례 제18조의2 (태양광 발전시설의 입지기준)"
- **수치 추출 난이도**: 조례마다 표현 다름 ("200m", "이백미터", "도로경계부터 200m")
  → LLM 파싱 85~90% 성공 + 20~30% 수작업 보정 필요

### 4. 건축물대장 API — 주택 군집 판별 (이미 보유)

- 이미 2차 개발로 키·검증 완료 → [api_건축물대장.md](api_건축물대장.md)
- `mainPurpsCdNm` = "단독주택" / "공동주택" 필터
- DBSCAN (eps=50m, minPoints=5) 으로 "5호 이상 밀집 군락" 판정

## 🏗️ 구현 알고리즘

### 태양광 설치여부 매칭

```
[축사/온실 리스트] ← 건축물대장 (2차)
        ∩
[태양광 허가 리스트] ← 태양광허가 API
        ↓
지번 매칭 or 좌표 반경 50m 내 매칭
        ↓
☀️ 설치완료 → 제외/흐림
⭕ 영업대상 → 표시
```

### 이격거리 판정

```python
def 이격거리_OK(지번: 좌표, 시군구):
    조례 = 조례DB.get(시군구)   # 기본값: 도로 200m / 주택 500m

    # 1. 도로 조건
    roads = 도로레이어.intersects(buffer(지번, 조례.도로_m))
    if any(r.lanes >= 2 for r in roads): return False

    # 2. 주택 군집 조건
    houses = 건축물대장.filter(주용도 IN ["단독주택","공동주택"]) \
                    .within(buffer(지번, 조례.주택_m))
    if DBSCAN(houses, eps=50m, min=5).has_cluster(): return False

    return True
```

> "지도맵 위에 태양광 그리기 / 패널수 / 시공견적 100kW당 1.3억 비율로 패널수 계산 /
> 20년 원리금상환 10년 아니면 완전 자기자본 부담 / 견적서가 나와야 합니다.

→ **영업 매출 직결 도구**. 1·2·3차 (탐색·검증) 의 종착점.

### 비즈니스 로직 (의뢰자 확정)

| 항목 | 값/방식 |
|---|---|
| 면적 → 패널 비율 | **2.5평당 1kW 1장** (고정) |
| kW 단가 | **130만원/kW** (= 1.3억/100kW, 변수 — 사용자 변경 가능) |
| 금리 | 변수 (사용자 입력) |
| 상환 옵션 | 자기자본 / 20년 / 10년 (3개 라디오) |
| SMP·REC 단가, REC 가중치, 일발전시간 | 모두 변수 (양식 PDF 분석으로 도출) |
| 효율 감소 | 첫해 0.99, 이후 매년 0.996 (양식 PDF 기준) |
| 유지보수율 | 연 3% (양식 PDF 기준) |

1. **견적서 PDF (A3 가로)** — 양식: [docs/예시/태양광 예상견적서(봉남리6-2) A3.pdf](예시/)
