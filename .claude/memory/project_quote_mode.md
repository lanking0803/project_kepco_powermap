## 1. 견적 모드 구조

```
/quote/[pnu] (풀스크린 라우트, 새 탭으로 진입)
  ├─ 상단바: [← 지도로] [ⓘ 부지 정보] 주소/PNU [PDF 출력]
  ├─ 좌측 패널 (320~384px): 5단계 워크플로
  ├─ 중앙: QuoteMap (카카오 위성 + 폴리곤 편집)
  └─ 우측 패널 (320~384px): 면적/평수 + 동별 상세 카드
```

5단계:
1. **영역 정의** — ✅ **완료** (2026-04-26)
2. **시설별 견적** — ✅ **완료** (2026-04-27, kW/시공비, 시설 4종 자동 추천 + 단가 인라인 편집)
3. **패널 시각화** — 🔜 **다음** (의뢰자 컨펌 완료 2026-04-27)
4. **배치도 PDF (A3)** — ⏳ (봉남리 양식)
5. **수지분석** — ⏳ (20년 시계열 + 3시나리오, 봉남리 PDF 검증 완료)

## 3. 코드만 봐선 알 수 없는 결정/맥락

### VWorld lt_c_spbd PNU 인덱스 결함 — BBOX 우회
- `fes:Filter PropertyIsEqualTo pnu=...` → 7000ms (풀스캔)
- `BBOX` → 44ms (spatial index 정상)
- 흐름: 필지 PNU → 필지 BBOX → BBOX 로 lt_c_spbd 호출 → 응답에서 pnu 일치만 클라이언트 필터
- 검증: [web/scripts/test-vworld-buildings/05-bottleneck.ts](../../web/scripts/test-vworld-buildings/05-bottleneck.ts)
- VWorld 측 결함이라 우회 외에 답 없음. lt_c_landinfobasemap (필지) 의 PNU 는 정상.

### 도로명주소 미부여 케이스 (대장 ⭕ + 폴리곤 ❌)
- 시골 동·식물시설(축사/온실 등) 은 건축물대장 등록되지만 도로명주소 미부여 → lt_c_spbd 폴리곤 0건
- 이런 부지는 ParcelInfoPanel "필지" 탭에서 ⚠️ 안내 자동 노출 (`polygonCount` prop 기반)
- 사용자는 견적 모드에서 [+ 영역 추가] 로 직접 그리기
- 검증 PNU: 충북 보은군 마로면 원정리 341-2 = `4372033030103410002`

### ParcelInfoPanel 통째로 재사용 — 새 컴포넌트 X
- 견적 모드 상단바 [ⓘ 부지 정보] = 메인 지도와 동일 컴포넌트
- 새 prop 2개만 추가: `polygonCount` (안내용), `inQuoteMode` (면적 산출 시작 버튼 숨김용)
- 데이터: atomic endpoint HTTP 캐시 hit 으로 사실상 0 fetch (capa 만 100ms)

### 영역 편집 UX 의뢰자 결정
- 점 갯수 자유 (4꼭지점 강제 X) — 자동 폴리곤 그대로 + 사용자 추가/삭제로 정밀화
- 점 추가 = 가장 긴 변 가운데 자동 (의도와 다른 위치는 드래그로 이동)
- 점 삭제 = 가장 평평한 점 자동 (모양 보존) + 마커 더블클릭은 정밀
- 영역 추가 = 부지 중앙 15m × 15m 사각형 → 사용자 드래그로 조정
- ＋ hover 마커는 깜빡임 이슈로 제거 (Q 폐기) → stepper 로 일원화
- 모든 기능은 우측 카드에 통일 (좌측은 워크플로 뼈대만)

### 카카오 SDK 한계 우회
- Polygon 자체 편집 기능 X → vertex 마다 별도 Marker (draggable)
- Marker 'drag' 이벤트 발화 X (dragstart/dragend 만) → requestAnimationFrame 폴링으로 실시간 표시
- 같은 패턴이 ✥ 핸들에도 적용
