> 4. D-3 임박 매물 빨강 깜빡임 강조 ✅

**의뢰자 의도** (2026-04-29 답변):

- 매물 발견 시 → 고객 카톡 전달 / 위치 메모 / 관심 등록 (관심등록은 1차 범위 X)

**검증 결과** (2026-04-28~29):
- ✅ 시도/시군구/동/면적/감정가/입찰기간 필터 작동 (`crawler/test_onbid_filter*.py`)
- ✅ 카테고리 소분류 `cltrUsgSclsCtgrId` 작동 (예: 10402=창고시설)
- ✅ 응답 `ltnoPnu` 19자리 100% 채움
- ✅ 상세 API `potoUrlList` = list (사진 URL = onbid.co.kr 공식 도메인)
- ⏳ 외부 링크 URL 패턴 / 유찰횟수 필터 = Phase 0 검증 잔여

**좌표 전략** (호출 부담 최소화):

- 같은 동 매물 = 카카오 클러스터러로 묶음
- 매물 클릭 시에만 (선택) PNU → VWorld 호출 + 캐시

**UI 우선 + Mock 데이터 전략 (2026-04-29 결정)**:
- UI 부터 mock 데이터로 흐름 완성 → 백엔드는 UI 가 필요한 데이터 모양에 맞춰 추가
- ⚠️ **개발 완료 시 mock 데이터/임시 코드 모두 제거** (사용자 명시: "덕지덕지 지저분 금지")
  - `web/lib/onbid/mock.ts` 삭제
  - 컴포넌트 안 mock import → 실 fetch 호출로 교체
  - "Mock 호환 분기" 흔적 코드 안 남김

**작업 분할** — UI 먼저
- Phase 0: 검증 마무리 (외부링크 URL 패턴, 유찰횟수)
- Phase 2 (UI): 타입+mock → [공매] 토글 → 검색 사이드바 → 매물 마커 → 클러스터러 → ParcelInfoPanel [공매] 탭 → 외부링크/즉석견적 → 모바일 (5~6시간)

- Phase 3: 시연 + 메모리 갱신
