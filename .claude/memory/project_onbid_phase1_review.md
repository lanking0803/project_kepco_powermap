# 공매 Phase 1 출시 후 재검토 항목

## 배경

2026-04-30 공매 기능 Phase 1 출시 완료 (커밋 `8d0ae36`). 출시 시점 기준 의뢰자 요구사항 100% 충족 + 영업 시선 흐름 UI 완성.

다음 항목들은 **출시 단계에서 함께 다루기엔 트레이드오프** 가 있어 보류. 의뢰자(조애옥 대표) 가 실제 영업에 사용 후 피드백을 받아 결정.

## 보류 항목

### 1. 공매 dedup 정책 ⚠️ (가장 중요)

**현재 동작** (커밋 `886fbc0`, `8d0ae36`):
- 같은 매물(cltrMngNo) 의 회차들 중 **첫 응답 row** 만 살림
- 캠코 응답 순서 = 가장 최신 회차(pbctNsq 큰 값) 가 먼저 → 가장 싼 가격 + 가장 먼 입찰일

**잠재 영업 신뢰 리스크**:
- 화면에 "D-5 · 622만원" 으로 표시되지만, **D-5 회차는 실제 622만원이 아님**
- 622만원은 마지막 6번째 회차 시나리오 (2026-08-26), D-5 는 첫 회차 (~7/22, 3,723만원) 일 가능성
- 영업이 622만원이라고 안내했는데 실제 입찰일에 가격 다르면 신뢰 손상

**선택지**:
- (A) 현재 그대로 — "최저 시나리오" 매력 강조
- (B) 임박 회차 우선 (cltrBidEndDt 최소) — D-day 정확
- (C) 회차 드롭다운/토글 — 영업이 직접 선택 (UI 복잡도 ↑)

**보류 이유**: 의뢰자가 실제 영업에서 "최저가 시나리오 매력" 을 어떻게 활용하는지 모름. 데이터 보고 결정.

**검증 방법**: 의뢰자에게 "지금 화면의 D-day 와 가격이 실제 캠코에서 진행 중인 회차와 일치하는지" 한 번 직접 비교 부탁.

---

### 2. 회차 드롭다운/토글

위 dedup 보강의 한 형태. 사용자 영업담당자가 회차별 가격 인하 시나리오를 보고 의사결정.

**보류 이유**: 영업 첫 인상의 깔끔함 우선. 의뢰자가 회차 정보를 실제로 쓰는지 모르고 추가하면 UI 복잡도만 ↑.

**검증 방법**: 의뢰자가 "이 매물 다른 회차도 보고싶다" 라는 요청을 하는지 모니터링.

---

### 3. 검색 결과 페이지네이션 / 더보기

**현재**: 캠코 응답 1,000건 cap, 회차 dedup 후 화면에 ~400매물 표시. 한도 초과 시 "결과가 너무 많아 일부 누락" 안내.

**잠재 이슈**: 의뢰자가 "전국 매물 다 보고 싶다" 또는 "내 시군구 매물 다 보고싶다" 라고 하면 누락된 매물이 영업 기회 손실.

**선택지**:
- 페이지네이션 (다음 페이지 버튼)
- 무한 스크롤
- numOfRows 더 키우기 (현재 1,000 cap, 캠코는 5,000 까지 안전 검증됨)

**보류 이유**: 안내 문구로 충분할 가능성. 의뢰자가 시군구까지 좁히면 누락 거의 없음.

---

### 4. 카테고리 분류 정교화

**현재** ([web/lib/onbid/categories.ts](../../web/lib/onbid/categories.ts)):
- 토지/유리온실/축사/창고/건물50plus 5종
- 캠코 sclsId 기반 매핑 + 매물명 키워드 후필터 (유리온실/축사 12100 공유)
- 300건 실측 분포 기반

**잠재 이슈**: 매물명에 "유리온실" 같은 키워드 안 적힌 진짜 유리온실은 분류 누락 가능. 의뢰자 핵심 카테고리(토지/유리온실/축사) 에서 빠지면 영업 기회 손실.

**검증 방법**: 의뢰자가 "이 매물 왜 안 떠요" 또는 "이게 왜 토지로 분류됐어요" 같은 피드백 시 분류 규칙 정교화.

---

### 5. 모바일 반응형 점검

**현재**: 데스크톱 위주 디자인. 영업이 외근 중 폰으로 매물 확인 시나리오 미검증.

**점검 대상**:
- OnbidVillageCard / OnbidVillageModal / OnbidTab 사진 갤러리 / 라이트박스
- Hero 한 줄 / 가격 카드 (그리드 3칸) 의 폭 깨짐 여부

**보류 이유**: 의뢰자 디바이스 사용 패턴 확인 후 우선순위 결정.

---

### 6. 임박 매물 알림

**현재**: 사용자가 직접 검색해야 D-day 확인 가능.

**제안**: D-1, D-3, D-7 임박 매물 자동 알림 (이메일/슬랙/푸시).

**보류 이유**: Phase 3 또는 별도 기능. Phase 1 단순 발견 도구로 충분.

---

### 7. 압류재산 추가 정보 (점유관계, 공매재산명세서)

**현재** 표시: 위치/접근성/활용/입찰조건/매수자격/납부사항/인도책임/기타사항/감정평가
**캠코 명세상 미사용 필드**:
- `ocpyRelList` (점유관계목록 — 소유자/임차인 등) — 압류재산 매수 시 핵심
- `papsInf` (공매재산명세서 — 처분청/체납 사유)
- `dsplVldCont` (매각유효내용)

**보류 이유**: 압류재산만 채워지는 옵션 필드라 비어있는 매물 다수. 의뢰자가 압류 매수가 본격 영업이면 추가 가치.

## 결정 트리거

각 항목은 **의뢰자가 영업 사용 중 다음 발화** 시 즉시 재검토:

| 항목 | 트리거 발화 |
|---|---|
| 1. dedup 정책 | "D-day 랑 가격이 캠코랑 안 맞네요" |
| 2. 회차 드롭다운 | "다른 회차 가격도 보고싶어요" |
| 3. 페이지네이션 | "내 지역 매물이 다 안 나와요" |
| 4. 카테고리 정교화 | "이 매물이 왜 카테고리에 없어요" / "왜 토지로 분류됐어요" |
| 5. 모바일 반응형 | "폰으로 보면 깨져요" |
| 6. 임박 알림 | "D-1 매물 자동으로 알려주세요" |
| 7. 추가 정보 | "점유관계 / 처분청 정보도 보여주세요" |

## 발화가 없으면

추가 작업 없이 현재 상태로 운영. **만들지 않은 기능 = 만들 가치를 검증하지 않은 기능**.
