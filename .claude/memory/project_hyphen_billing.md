# Hyphen 경매다 API — 비용 구조와 운영 모드 전환

## 결제 모델 (실측 + 공식 페이지 검증 2026-05-02)

```
TR슬림 (의뢰자 선택)
├ 멤버십 월정액: 10만원 (VAT 포함 결제 시 11만원)
│  → 충전한 비즈머니에서 차감 (별도 결제 X — 처음에 의뢰자가 헷갈렸던 부분)
│  → 유효기간 1개월 (예: 2026-05-02 ~ 2026-06-01)
│  → 갱신 안 하면 errCd=HDM006/HDM009 응답 → UI 가 결제 안내 배너 표시
├ 호출당 종량: "건 별 정상요금"
│  → 페이지 어디에도 단가 미공시 ⚠️
│  → 호출 후 사용량 리포트로만 확인 가능
│  → 단가 알려면 1600-4173 통화 필요 (5/2 미실시)
├ 무료 테스트베드: Hyphen-Gustation: Y 헤더 부착 시
│  → 비즈머니 차감 X, 20초/호출 레이트리밋 (HDM016)
│  → 페이지에 "10건 무료 테스트" 표기 (실측: 30번 넘게 호출했는데 비즈머니 그대로)
│  → 즉 테스트베드 호출은 사실상 무제한 (10건은 운영 모드 무료분일 가능성)
└ 보너스 캐시 사용 불가 (제휴 상품)
```

**Why**: 의뢰자가 "비즈머니 11만원 충전 = 결제 끝" 으로 오해함. 실제로는 멤버십 가입(상품 신청) 별도 필요. 비즈머니는 "선불 지갑" 일 뿐.

**How to apply**: 의뢰자에게 비용 안내할 때 "11만원 멤버십 = 충전한 비즈머니에서 자동 차감" 으로 명확히 설명. 별도 결제로 오해하지 않도록.

## errCd 분류 (실측 검증)

| errCd | 의미 | apiStatus | UI 배너 |
|---|---|---|---|
| 200 | 정상 | ok | 없음 |
| 407 | 매물 0건 (정상 응답) | empty | 없음 |
| HDM006 | UserId/Hkey 헤더 누락 | auth_failed | 🔴 결제 만료 안내 |
| HDM009 | UserId/Hkey 값 오류 (변조/만료) | auth_failed | 🔴 결제 만료 안내 |
| HDM012 | 권한 없음 (운영 모드 미신청 상태) | no_permission | 🔴 멤버십 가입 안내 |
| HDM016 | 테스트 모드 20초 레이트리밋 | rate_limited | 🟡 잠시 후 재시도 |

**미확인 errCd**: 비즈머니 부족 / 사용량 한도 초과 등 — 운영 모드에서 발견 시 추가 등록 필요.

## 우리 코드의 모드 토글 (운영/테스트)

`web/lib/hyphen/client.ts` `authHeaders()` 함수:

```typescript
// 기본값 = 테스트 모드 (Hyphen-Gustation: Y 부착, 무료, 20초 레이트리밋)
// 환경변수 HYPHEN_OPERATION_MODE=Y 면 운영 모드 (헤더 미부착, 비즈머니 차감)
if (process.env.HYPHEN_OPERATION_MODE !== "Y") {
  headers["Hyphen-Gustation"] = "Y";
}
```

**Why**: 의뢰자가 멤버십 미가입 상태에서 운영 모드로 전환하면 HDM012 무한 응답 → 영업 사용 불가. 안전 기본값 = 테스트 모드.

**How to apply**:
- 의뢰자가 멤버십 가입 + 운영 권한 확인 후에만 `HYPHEN_OPERATION_MODE=Y` 환경변수 추가
- `.env.local` (개발) + Vercel 환경변수 (배포) 둘 다 등록
- 추가 후 dev 서버/Vercel 재배포 필요 (런타임 적용)

## 의뢰자 결정 (2026-05-02 카톡)

**1번 선택 — 테스트모드 시연 → 사용해보고 2번(멤버십 결제) 으로 전환**

> "1.번입니다. 10만원 충전은 해놓은상태입니다 네 좀사용햐보고 2번으로가겠습니다"

→ 의뢰자가 충전한 11만원 비즈머니는 그대로 둠 (멤버십 가입 시 자동 차감 예정).
→ 영업 사용성 점검 후 의뢰자 본인이 멤버십 가입 결정.

## 의뢰자 멤버십 가입 결정 시 우리가 할 일 (체크리스트)

1. **단가 확인** (필수, 결제 전):
   - 1600-4173 메뉴 2번 통화 또는 help@hyphen.im 메일
   - "TR슬림 부동산 법원경매(경매다) 호출당 단가" 질의
2. **의뢰자 멤버십 결제** (의뢰자 본인이 사이트에서 직접):
   - https://hyphen.im/mypage/my-membership → "결제하기" 클릭
3. **운영 권한 확인**:
   - 멤버십 가입 후 errCd=HDM012 사라지는지 확인 (1번 호출 테스트)
   - 사라지지 않으면 별도 신청 필요 (고객센터 문의)
4. **운영 모드 전환** (코드 측):
   - `web/.env.local` 에 `HYPHEN_OPERATION_MODE=Y` 추가
   - Vercel 환경변수에도 동일 추가
   - 둘 다 dev/prod 재시작 필요
5. **단가 검증** (1~2건 호출 후):
   - 마이페이지 → 비즈머니 잔액 변화로 단가 역산
   - 우리 코드의 30분 캐시/20페이지 cap 정책 검토 (단가 비싸면 강화)
6. **expiry 필드 갱신**:
   - `web/app/admin/api-manager/_lib/services/hyphen.ts` 의 `expiry` (현재 2026-06-02)
   - 의뢰자 결제일 기준 1개월 후로 수정
   - SECRETS.local.md 의 만료일도 동기화

## 단가 확인 통로 (3가지)

1. **고객센터 1600-4173** (메뉴 2번, 평일 08:30~17:30) — 가장 빠름
2. **이메일 help@hyphen.im** — 1~2일 회신 가능성
3. **결제 후 호출 1건 → 비즈머니 차감액 = 단가 역산** — 즉시지만 환불 불가 (호출 발생 후)
