# 💰 SUNLAP 결제 기록

## 🧾 총 금액

- **개발비 합계**: **1,410만** (1차 710 + 2차 700)
- **인프라 1회성 20만**: **무상 서비스** (수집기 3→5 증설 + 도메인 연결 포함)
- **인프라 월 운영비**: 월 5만 (별도 청구)

## 📅 결제 조건 (의뢰자와 합의)

- **선금 200만** (2026-04-21 수령 완료) → 개발 착수
- **중간정산 500만** (2026-05-02 수령 완료) ✅ — 합의일(5/6)보다 4일 앞당겨 입금
- **잔금 (1,410 − 200 − 500 = 710만)**: 2차 영업 발굴 시스템 + 경매 완료 후
- **인프라 20만 (1회성)**: 서비스 처리로 받지 않음

## ✅ 수령 내역

| 일자 | 금액 | 항목 | 비고 |
|---|---|---|---|
| 2026-04-21 | 200만 | 개발 선금 | 전체 1,410만 중 선금 |
| 2026-04-21 | 5만 | 서버 운영비 (1개월) | **다음 결제일: 2026-05-21** |
| 2026-05-02 | 500만 | 중간정산 | 합의일 5/6 → 4일 조기 입금 |

**누적 수령**: 705만 (개발비 700 + 서버비 5)
**개발비 잔액**: 710만 (1,410 − 700)

## ⚠️ 운영 메모

- 서버 운영비는 "사용자 수 증가, 신규 인프라 추가 시 변동 가능" 이라 카톡 고지됨
- 매월 청구 리마인드 필요 (의뢰자가 먼저 챙길 가능성 낮음)
- 서버비 연체 시 Supabase Pro 해지 → 서비스 중단 리스크 있음 → 최소 1주일 전 리마인드 권장

**Why:** 선금 방식 + 완성 후 잔금이라 진행 중엔 개발비 현금 흐름 0. 서버비 월 5만만 정기 유입되므로 누락 없이 챙겨야 함. 의뢰자는 개발에 집중하고 있어 운영비 청구는 개발자가 주도해야 함.

**How to apply:**
- 매월 21일 근처에 의뢰자에게 서버비 안내 카톡 발송
- 수령 후 이 파일 표에 추가 + 잔액/누적 체크
- 개발 완료 시점 확정되면 잔금 청구 문구 별도 준비
