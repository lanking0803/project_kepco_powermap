# 채택 안 한 대안 (참고)

원래 메모는 **"DB 에 시군/읍면 집계 MV 추가 + 새 endpoint"** 방향이었음.
실제로는 **클라이언트 즉석 GROUP BY** 채택 — 이유:
- DB/API 변경 0 (단일 파일 수정)
- 빈값/동명 폴백을 코드에서 일반 로직으로 처리 가능
- bounds 필터 후 그룹핑이라 데이터 작음 (수~수백 그룹) — 즉석 연산 충분히 빠름

# 핵심 함정 (구현 시 부딪힌 것)

1. **CSS transform: translate(-50%, -50%)** + 카카오 SDK xAnchor/yAnchor 이중 보정 → 마커가 화면 밖으로 밀려남.
   해결: SDK 의 xAnchor 0.5 + yAnchor 0.5 만 쓰고 CSS translate 제거. (.kepco-card-marker L80 주석에 같은 함정 경고 있음)

2. **빈 string `||` 폴백 누락** — 세종 sep_3 = `""` (null 아닌 빈 string) 케이스에서 `addr_si || ""` 가 빈값 반환. `pick()` 헬퍼 (trim().length > 0 검사) 로 일반화.

3. **clustererRef 가드** — 클러스터러를 안 쓰는데 props effect 에 가드가 남아있어 첫 rebuild 가 호출 자체 안 되는 케이스. 가드 제거 필요.
