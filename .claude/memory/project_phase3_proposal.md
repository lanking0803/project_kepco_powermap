### 요청 8 (2026-05-04 추가): 필지(건축물대장) 정보 수집기 — 시군구 단위 검색 지원
- **트리거**: 의뢰자 사용자 피드백 — "시군구 단위로만 선택하고 추출하고 싶다" (2026-05-04 카톡)
- **데이터 제공처 한계 확인됨**: 국토부 건축HUB API 는 정책상 **읍·면·동(리) 단위 필수** — `bjdongCd` 빈값 시 응답 0건. 우회 불가 ([reference_bldg_register_api.md](reference_bldg_register_api.md))
- **유일한 해결**: KEPCO 수집기 패턴 미러로 **사전 적재 + Supabase 조회**
- **재사용 자산**:
  - `crawler/` Python 인프라 (작업 큐 / 2중 제어 모델 / 장애복구) 그대로 활용
  - `bjd_master` 좌표 보강 패턴 ([reference_supabase_rpc_plan_trap.md](reference_supabase_rpc_plan_trap.md))
  - `enrichFacilities` 분류·평수 계산 로직 ([web/lib/facility/enrich.ts](../../web/lib/facility/enrich.ts))
- **적재 규모 (실측 기반 추정)**:
  - BJD 약 22,000개 × 평균 3페이지 = 약 66,000회 외부 호출
  - 일일 한도 10,000회 → **약 5일 분할 적재** (단일 워커 + 500ms 직렬화 기준)
  - DB 약 700만 행 / 압축 후 3~5GB → Supabase Pro 플랜 (월 25$ 약 3.3만원) 필요
- **갱신 정책**: 분기 1회 전체 재적재 (건축물대장 변동률 < 0.1%/월, 매일 갱신 불필요)
- **사용자 효과**:
  - 현재: 동/리 단위만 가능, 7~10초 응답
  - 적재 후: 시도/시군구/동/리 자유 선택, 0.5초 응답 (DB SELECT 만)
  - 동시 사용자 무제한 (외부 API 한도 무관)
- **견적 영향**: 단독 작업, **+250~350만 / 2~3주** 예상
  - 적재 모듈 신설 (KEPCO 인프라 미러) ~1주
  - DB 스키마 + 마이그레이션 + 인덱스 ~3일
  - 1회성 전국 적재 + 검증 ~1주
  - Supabase Pro 플랜 운영비 별도 (월 3.3만원, 의뢰자 부담)
- **3차 안내 시 옵션화 권장**:
  - (a) 풀 스펙 — 전국 즉시 적재, 시군구/시도 검색 지원
  - (b) 단계적 — 영업 권역 시도부터 우선 적재 (비용 절반)
  - (c) 보류 — 현재 동/리 단위로 충분 시 적재 안 함

⚠️ **현재 상태**: 의뢰자 결정 — "3차 개발건으로 빼고, 3차 견적 안내 시 정확히 산출해서 함께 안내" (2026-05-04)

### 요청 9 (2026-05-08 추가): 마을·지번 즐겨찾기 — 그룹 기반 관리
- **트리거**: 의뢰자 카톡 (2026-05-08) — "즐겨찾기 그룹을 만들어서 그룹 내에서 마을이나 지번을 추가할 수 있는 기능"
- **핵심 의도**: 영업담당자가 즐겨찾기 그룹(예: "광주권 진행중", "전남 유망", "보류") 을 만들고, 마을/지번을 해당 그룹에 분류해 관리
- **사용 흐름**:
  1. 마을 마커 또는 지번 패널에서 [⭐ 즐겨찾기 추가] 버튼
  2. 그룹 선택 모달 (기존 그룹 선택 / 신규 그룹 생성)
  3. 선택한 그룹에 등록
  4. 사이드바 "즐겨찾기" 탭에서 그룹별 목록 조회 + 클릭 시 해당 위치로 이동
- **기술 구조 (의뢰자 인지)**:
  - **별도 DB 테이블 필수** — 계정마다 즐겨찾기 다르게 지정해야 함 (user_id 기반)
  - 테이블 2종 예상: `favorite_groups` (사용자별 그룹) + `favorite_items` (그룹-마을/지번 매핑)
  - 항목 type = `village(BJD)` / `parcel(PNU)` 분기
- **기존 요청 4 (경공매 관심물건) 와의 관계**: 통합 vs 분리 의뢰자 결정 필요
  - 통합 시: type 에 `onbid` / `court` 추가 → 한 시스템에서 4종(마을/지번/공매/경매) 관리
  - 분리 시: 본 요청은 마을/지번만, 경공매는 별도 시스템
- **견적 영향**: 단독 작업, 의뢰자 결정 후 산출

⚠️ **의뢰자 지시 (2026-05-08)**: "3차 개발내용에 추가만 해줘. 고객한테 보낼 메세지는 필요하면 말할게" — 메모리 갱신만 진행, 카톡 안내는 의뢰자 요청 시.

## ✅ 검증 결론 (2026-04-17)

**전부 외부 데이터 연계 가능**. 조례 수치 추출만 LLM 반자동.
- 태양광 허가 API · 도로 SHP · 건축물대장 · 법제처 조례 API 모두 무료 공공 데이터
- 한계: 3kW 이하 자가발전 누락(허가 대상 아님 — 축사/온실은 대부분 30kW↑라 영향 적음)

## 💰 견적 초안 (의뢰자 확답 전)

- 단독 진행: **약 550만 / 10~12주** (①설치여부 + ②이격거리 옵션 B 또는 D 추천)
- 2차와 묶으면 건축물대장/VWorld 중복 제거로 100~150만 절감 가능
- 옵션 B(LLM 자동 파싱) vs D(원문 링크) — D는 유지보수 부담 0, B는 정확도↑ but 연 50~100만 유지보수
- **요청 8 (필지 수집기) 추가 시**: +250~350만 / +2~3주 + Supabase Pro 운영비 월 3.3만원 (의뢰자 부담)

**경쟁력**: 해줌/솔라커넥트/서니로직 **아무도 이격거리 필터 없음** → 강력한 차별화

## 🚦 현재 상태 (2026-05-05 갱신)

**1·2차 완료 보고 + 3차 개발 7항목 안내 카톡 발송 완료. 의뢰자 답장 대기.**

발송한 3차 개발 7항목 (의뢰자 멘트):
1. 이격거리 조건 충족 부지 (도로·주택 거리 / 시도 조례 반영) — 요청 2
2. KEPCO 수집기 17개 시도 전체 확장 — 요청 3
3. 경·공매 관심물건 등록 (즐겨찾기) — 요청 4
4. 견적 관리 (목록·상태·이력) — 요청 5
5. 견적 일발전시간 위치 기반 자동 추천 — 요청 7
6. 필지 수집기 개발 — 시군구 단위 검색 지원 — 요청 8
7. 시뮬레이터 + 견적서·배치도 PDF 자동 생성 — Phase 4 통합

**제외된 항목**:
- 요청 1 "태양광 설치여부 표시" — 의뢰자가 "입지탭에 이미 구현"으로 인식. 중복 안내 회피 차원에서 카톡 미포함. 추후 의뢰자가 다시 거론하면 입지탭 기존 기능 재안내 + 마커 색상 분기 추가 작업으로 재제안 가능.
- 요청 6 "경매 지도 모드" — 2차에서 흡수 완료 (경매 모드 라이브)

**다음 단계**:
1. 의뢰자 답장 받으면 → 잔금 입금 일정 / 로고 PNG 수령 / 3차 진행 의향 확인
2. 3차 진행 OK 시 → 7항목 옵션화 견적 안내 (단독 550만 + 필지수집기 250~350만 등)
3. 견적 확정 후 착수
