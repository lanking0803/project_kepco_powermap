# 공매 + 경매 통합 (2차 영업 발굴 시스템)

## 의뢰자 의도 (2026-04-25 확정)

> **"태양광 적합 부지를 시세보다 싸게 매입할 기회를 자동 발굴하는 채널"**

- 의뢰자(솔라엘디 안홍열 대표)가 직접 인정한 의도
- 단서 4개 정황 분석 후 의뢰자가 컨펌 (대화 채널: 개발자 직접 확인)
- 즉 단순 부가정보가 아니라 **토지 매입 의사결정 도구**의 핵심 축

**Why**: 태양광 사업자에게 "좋은 입지의 토지를 저가로 확보"는 사업성 직결. 공시지가/실거래가(시세 추정) + 공매/경매(저가 매물 = 매입 기회) 가 한 묶음으로 토지 가치 평가/매입 결정 패키지를 구성.

**How to apply**:
- UI 설계 시 "토지 가치 정보" + "매입 기회" 두 축으로 묶어 제시
- 공매/경매 매물 표시 시 **시세 대비 할인율** 강조 (영업 임팩트 핵심 지표)
- KEPCO 여유선로 ∩ 공매·경매 결합 필터 = 차별화 포인트 (경쟁사 불가)

## 범위 (2개 데이터 모두 개발)

- 견적서: [docs/견적_1차2차.md](../../docs/견적_1차2차.md) 2차 4번
- 공식 API:
  - 목록: `getRlstCltrList2` (https://apis.data.go.kr/B010003/OnbidRlstListSrvc2)
  - 상세: `getRlstDtlInf2` (https://apis.data.go.kr/B010003/OnbidRlstDtlSrvc2)
- 활용신청: 2026-04-25 완료, 자동승인. 공공데이터포털 동일 인증키 (건축물대장과 공유)
- 명세 docx → 텍스트: [docs/api_specs/온비드_공매/_extract.txt](../../docs/api_specs/온비드_공매/_extract.txt) / [_extract_상세.txt](../../docs/api_specs/온비드_공매/_extract_상세.txt)
- 검증: 응답에 `ltnoPnu` 19자리 PNU 직접 제공 → VWorld 폴리곤 그대로 결합 가능
- 호출 테스트: [crawler/test_onbid.py](../../crawler/test_onbid.py)
- 현재 압류재산만 30,178건 진행 중

- API/크롤링 전략은 별도 논의 후 결정

## UI 통합 전략 — 가격탭 복잡도 관리

이전 [docs/api_endpoints.md:61](../../docs/api_endpoints.md#L61) 에서 공매가 "가격 탭" 으로 사전 분류돼 있었음. 사용자(개발자) 가 의뢰자 의도 확인 후 가격탭에 통합하는 방향이 더 자연스럽다고 판단. 단 가격탭 복잡도 우려 있음 ([web/components/map/ParcelInfoPanel.tsx](../../web/components/map/ParcelInfoPanel.tsx) 의 가격탭은 이미 토지/건물 sub-tab 보유).

권장 구조 (잠정, 추후 의뢰자/UI 검토 필요):
```
가격탭
├── 📍 이 지번 — 공시지가/추정가
├── 📊 시세       — 토지/건물 실거래가 (sub-tab, 이미 구현)
└── 🏛 매입 기회   — 공매/경매 (sub-tab, 신규)
       ├── 이 필지가 매물이면 배지 표시
       └── 인근 매물 리스트
```

+ **지도 레이어 별도** (영업 발굴 모드):
- 우상단 토글 [공매·경매 🏛] → 전국 매물 핀
- viewport-aware 마커 (KEPCO 패턴 재활용)
- 필터: 재산유형 / 할인율 / 면적 / KEPCO 여유 결합

즉 가격탭(검증) + 지도 레이어(발굴) **둘 다 필요** — 데이터·API 1번 만들고 표시만 두 곳.
