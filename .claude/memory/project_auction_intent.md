# 공매 통합 (2차 영업 발굴 시스템) — 경매는 포기됨

## 의뢰자 의도 (2026-04-25 확정)

> **"태양광 적합 부지를 시세보다 싸게 매입할 기회를 자동 발굴하는 채널"**

- 의뢰자((주)한국에텍 조애옥 대표)가 직접 인정한 의도
- 단서 4개 정황 분석 후 의뢰자가 컨펌 (대화 채널: 개발자 직접 확인)
- 즉 단순 부가정보가 아니라 **토지 매입 의사결정 도구**의 핵심 축

## 표시 대상 카테고리 (2026-04-26 의뢰자 명시)

**포함 (5종)**:
- 토지
- 유리온실
- 축사
- 건축물 (50평 이상 = 165㎡↑)
- 창고

**제외**:
- 아파트 (주거시설)
- 동산 (가전·집기 등)
- 자동차

→ 2차 1번 항목 "유리온실/축사 + 일반 건물 평수 구간" 과 동일 카테고리. 의뢰자 머릿속에서 **공매는 별개 기능이 아니라 2차 1번 발굴 시스템의 데이터 채널**.

**캠코 필터 매핑**:
- `cltrUsgLclsCtgrId = 10000` (부동산만)
- `cltrUsgSclsCtgrNm` 으로 아파트 소분류 제외
- 50평 = `bldSqms >= 165` 추가 조건
- 정확 코드 매핑은 [docs/api_specs/온비드_공매/_extract.txt](../../docs/api_specs/온비드_공매/_extract.txt) 코드표 + 호출 검증 필요

**Why**: 태양광 사업자에게 "좋은 입지의 토지를 저가로 확보"는 사업성 직결. 공시지가/실거래가(시세 추정) + 공매/경매(저가 매물 = 매입 기회) 가 한 묶음으로 토지 가치 평가/매입 결정 패키지를 구성.

**How to apply**:
- UI 설계 시 "토지 가치 정보" + "매입 기회" 두 축으로 묶어 제시
- 공매/경매 매물 표시 시 **시세 대비 할인율** 강조 (영업 임팩트 핵심 지표)
- KEPCO 여유선로 ∩ 공매·경매 결합 필터 = 차별화 포인트 (경쟁사 불가)

## 범위 (2개 데이터 모두 개발)

### 공매 (100만, 청구)
- 출처: 캠코 온비드 (공공데이터포털)
- 견적서: [docs/견적_1차2차.md](../../docs/견적_1차2차.md) 2차 4번
- 공식 API:
  - 목록: `getRlstCltrList2` (https://apis.data.go.kr/B010003/OnbidRlstListSrvc2)
  - 상세: `getRlstDtlInf2` (https://apis.data.go.kr/B010003/OnbidRlstDtlSrvc2)
- 활용신청: 2026-04-25 완료, 자동승인. 공공데이터포털 동일 인증키 (건축물대장과 공유)
- 명세 docx → 텍스트: [docs/api_specs/온비드_공매/_extract.txt](../../docs/api_specs/온비드_공매/_extract.txt) / [_extract_상세.txt](../../docs/api_specs/온비드_공매/_extract_상세.txt)
- 검증: 응답에 `ltnoPnu` 19자리 PNU 직접 제공 → VWorld 폴리곤 그대로 결합 가능
- 호출 테스트: [crawler/test_onbid.py](../../crawler/test_onbid.py)
- 현재 압류재산만 30,178건 진행 중

### 경매 ⏳ 의뢰자 API 키 발급 대기 (2026-05-01 재개 합의)

- **2026-05-01**: 의뢰자가 시연본 보고 "경매도 같이 진행해야할 것 같다" 추가 요청 → 재개 합의.
  - 개발자 답변: "이전에 알려드린 사이트(Hyphen)에서 API키 발급받아 전달해주시면 개발 가능"
  - 의뢰자 답변: "API키 발급해서 주기로 했어. 발급되면 말해줄게"
  - 즉 **의뢰자가 직접 Hyphen 가입 + 월 10만 부담 + API 키 전달 → 개발 착수 = 무상 서비스**

  - 사법정보공유포털 (대법원 공식): 부동산 경매 미제공 (민사/독촉/회생파산만)
  - CODEF 경매사건검색 API: 단건 조회만 가능 (목록 sweep 불가)
  - 법원경매정보 사이트 직접 호출: 기술적으로 가능 (cookieless POST OK, [crawler/](../../crawler) test 검증) 그러나 **약관 위반 + 차단 위험** 으로 차선책 보관
- 우리 계정: CODEF 가입만 완료 (hicor@naver.com, 미사용 / [docs/SECRETS.local.md](../../docs/SECRETS.local.md))

## UI 통합 전략 — **2차 1번 발굴 시스템에 흡수 확정 (2026-04-26)**

의뢰자 컨펌 완료: "**공매 매물을 2차 1번 화면(시설 위 깜빡이 원)에 같이 표시 + 공매면 색상 구분**" → "네 좋습니다"

### 통합 구조
```
2차 1번 영업 발굴 시스템 (유리온실/축사+평수 250만 + 공매 100만)
└── 지도에 시설 위 깜빡이는 원 마커
     ├── 🔵 일반 (KEPCO 여유선로 충분)
     ├── 🔴 KEPCO 부족
     └── 🟥 공매 진행 중 ⭐ (NEW — 시설이 공매 매물이면 우선 색상)

표시 카테고리:
  ✓ 토지 / 유리온실 / 축사 / 창고 / 건축물 50평+

제외:
  ✗ 아파트 / 동산 / 자동차
```

### 데이터 결합
- 캠코 공매 응답의 `ltnoPnu` (PNU 19자리) → 우리 건축물대장/필지 데이터와 PNU 조인
- 동일 PNU 가 공매 매물이면 시설 마커에 공매 표시 우선 적용
- 별도 토글/별도 화면 없음 — 2차 1번 화면 1개로 통합

### 가격탭 배지 (보너스, 의뢰자 미언급이지만 자연스러운 추가)
- 사용자가 필지 클릭 → 가격탭 → 그 필지가 공매 매물이면 작은 배지 1줄 자동 표시
- 데이터·API 동일 (재사용)
- 의뢰자 추가 컨펌 필요 X (자연스러운 보조 기능)
