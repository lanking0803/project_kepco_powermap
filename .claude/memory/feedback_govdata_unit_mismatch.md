# 정부 공공 데이터 — 행정구역 단위 가정 금지

## 사고 (2026-05-02)
- 자연취락지구 검색 시 천안시 동남구(44131)/서북구(44133) → 0건
- 직전 답변: "도시화돼서 자연취락지구 없을 만함" → **추측, 틀림**
- 실 검증: 천안시 시 단위(44130)에는 50건 등록. 일반구 단위로는 미등록.
- 영향: 천안/수원/창원/안산/안양/고양/용인/청주/포항/전주 등 일반시 일반구 검색 시 결과 누락
  + 강남구/종로구 같은 광역시 자치구는 자연취락지구 0 + 집단취락지구만 응답 (영업 무관)

## Why (왜 가정 금지)

bjd_master 는 **행안부 표준 코드** — 일반시 일반구도 5자리로 명확히 구분(천안시 동남구=44131).
하지만 외부 정부 API(VWorld 등)는 **데이터 등록 시 그 표준을 일관성 있게 따르지 않는다**:
- 어떤 시는 시 단위로만 등록 (천안/수원/창원)
- 어떤 시는 구 단위로만 등록 (분당구는 시 단위에 없는 데이터가 분당구 단위에 있음)
- 어떤 광역시 자치구는 다른 용도지구만 등록 (강남구는 집단취락지구만)

같은 typename·같은 응답 구조여도 **시군구별로 등록 단위가 다르다**.

## How to apply

### 새로운 정부 API 사용 시 절차

1. **bjd_master 표준 단위와 일치한다고 가정 금지** — 첫 줄
2. **실 호출 검증**:
   - 광역시 자치구 (예: 11680 강남구)
   - 일반시 시 단위 (예: 44130 천안시)
   - 일반시 일반구 (예: 44131 동남구)
   - 일반 군 (예: 46720 곡성군)
   - 광역시 군 (예: 28710 강화군)
3. 케이스별 응답 패턴 파악 후 호출 전략 결정 (단일 호출 / 다중 호출 / 합치기)
4. 추가로 응답 자체에 **유사 데이터 혼재 검사** (자연·집단취락지구 혼재 같은)

### 반드시 검증할 것

- `count > 0` 인지뿐 아니라 **응답의 sigg_name / uname 등 라벨 종류**
- 시 단위 호출과 구 단위 호출의 **합집합/교집합** 관계 (성남 분당구 = 둘 다 따로)
- 응답 라벨 텍스트의 정확성 (예산군 호출이 "부여군" 으로 응답되는 사례)
